10-14日 论文50%攻击

简要讲述一下我这个方法现在遇到的问题：

由于 CRH 公式如下：

$$w_s = log(sum(所有人的误差) / s的误差)$$

误差表示的是真值和工人数据的距离。当坏人数量恰好为 50% 时，就会发生如下情况：

$$
| 0 | | | T | | | 1 |
$$

  • 0 表示好人的数据围绕的区域
  • 1 表示恶意数据围绕的区域

在这种情况下，真值大致会分布在中间。因此，我的创新点——通过以下公式来鉴别其他工人的数据是否合理：

$$
degree[s] = e^(W_0 - W_s) = 好人的误差 / s的误差
$$

在这种情况下效果会变得很差。可以明显看到：

| 0 | | | T | | | 1 |

好人距离真值 (T) 3 格，坏人距离真值 (T) 也是 3 格，这就是我遇到的问题。

又因为我的整个模型要保证隐私，这让我困扰了很久。

我想到的解决方案

引入百万富翁比大小机制。通过该机制，两个服务器拥有不同的权重，S_0 拥有真值 S_1。该机制能够安全地执行 T - b_s 和 X_s - b_s 的比大小。

  • S_0 获得 T 和 X_s 的大小情况后，在迭代完成后通知 TA，让 TA 进行筛选。

筛选策略

筛选策略采用了一种启发式方法。因为只有 TA 拥有可信工人是谁的信息，所以 TA 需要额外记录每个工人提交任务与可信工人相同的次数。设置一个阈值，当相同的次数比率小于 1/7 时，直接将该工人的权重归零。

设置原因

  • 在真值准确的情况下，好人有 0.0078 的概率小于这个阈值，这是一个几乎不可能的事件。
  • 而在遇到 50% 攻击的情况下，坏人会踩雷。
